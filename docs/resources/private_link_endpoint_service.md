# velodb_private_link_endpoint_service (Resource)

Use the *velodb_private_link_endpoint_service* resource to register an **outbound** PrivateLink endpoint service — a service that VeloDB's infrastructure should be able to connect TO (e.g., a private API your application exposes to VeloDB clusters for data lookup).

Not to be confused with the warehouse's **inbound** PrivateLink service (which is auto-provisioned by VeloDB and managed via [`velodb_warehouse_private_endpoint`](./warehouse_private_endpoint.md)).

## ⚠️ Known issue — endpoint is non-functional in sandbox

~> **This resource cannot currently be used against the public sandbox.** `POST /v1/private-link/endpoint-services` returns HTTP 500 for all cloud providers (`aws`, `aws-cn`, `aliyun`) in the sandbox environment. The provider code is correct per the OpenAPI spec, but the sandbox implementation is incomplete. Reported as API issue.

| Operation | Sandbox status |
|---|---|
| Create (`POST`) | ❌ Returns 500 `InternalError` for all cloud providers |
| List (`GET`) | ✅ Works (returns `[]` if none registered) |
| Get by ID (`GET`) | Unknown — nothing to read |
| Update description (`PATCH`) | Unknown |
| Delete (`DELETE`) | Unknown |

Until the sandbox is fixed, leave this resource out of your configs.

## Example Usage (reference — for future use)

```terraform
resource "velodb_private_link_endpoint_service" "corp_api" {
  cloud_provider        = "aliyun"    # aliyun | tencent_cloud | hwcloud | aws-cn
  region                = "cn-beijing"
  zone                  = "cn-beijing-a"
  endpoint_service_name = "cn-beijing-vpce-svc-abc123"
  provider_account_id   = "1234567890"
  description           = "Corporate API gateway"
}

output "service_id" {
  value = velodb_private_link_endpoint_service.corp_api.id
}
output "connected" {
  value = velodb_private_link_endpoint_service.corp_api.connected
}
```

## Schema

### Required

- `cloud_provider` (String) Cloud provider. One of `aliyun`, `tencent_cloud`, `hwcloud`, `aws-cn` (per spec). Changing forces new resource.
- `endpoint_service_name` (String) Cloud-side endpoint service name. Changing forces new resource.
- `region` (String) Cloud region. Changing forces new resource.

### Optional

- `description` (String) Service description. Updatable in place.
- `endpoint_service_id` (String) Cloud-side endpoint service ID. Some providers can infer from `endpoint_service_name`. Changing forces new resource.
- `provider_account_id` (String) Cloud account ID that owns the endpoint service. Required by some providers. Changing forces new resource.
- `zone` (String) Availability zone hint (provider-dependent). Changing forces new resource.

### Read-Only

- `connected` (Boolean) Whether the service is currently connected.
- `created_at` (String) Registration time in RFC 3339 format.
- `id` (String) Endpoint service identifier assigned by VeloDB.
